PROMPT 4:
"How should the scoring formula work?"

AI RESPONSE SUMMARY:
- I implemented the design document formula:
  Correctness * 1000 * (1 - elapsed / total * 0.5)
- I used server-side `clock.instant()` for submitted time.
- I gave wrong answers zero points.
- I calculated scores when the question closes instead of on every submission.

WHAT I LEARNED:
- Batch scoring on question close keeps the leaderboard consistent for everyone.
- Speed-based scoring gives faster correct answers more points.
- The server must clamp elapsed time so late or weird timestamps cannot produce
  negative or impossible scores.

CODE SNIPPET USED:
```java
private int pointsFor(CurrentQuestion current, Instant submittedAt) {
    long elapsedMs = Math.max(0, Duration.between(current.startedAt, submittedAt).toMillis());
    long totalMs = Math.max(1, Duration.between(current.startedAt, current.endsAt).toMillis());
    double elapsedRatio = Math.min(1.0, elapsedMs / (double) totalMs);
    return (int) Math.round(1000.0 * (1.0 - elapsedRatio * 0.5));
}
```

ISSUES ENCOUNTERED:
- Scoring based on client-provided timestamps would allow cheating.
- Scoring immediately on answer submission would make partial leaderboards
  visible before the question ended.

HOW I FIXED IT:
- I used only the server clock for answer timestamps.
- I stored submitted answers first, then processed all answers when the question
  timer expired.

TESTS ADDED:
- Verifies that a correct answer after 5 seconds on a 20 second question earns
  875 points.
- Verifies that wrong answers receive 0 points.

PROMPT 5:
"How do I make the leaderboard sorted correctly and show final winners?"

AI RESPONSE SUMMARY:

- I used correct answer count as a tie-breaker.
- I used case-insensitive nickname order as a final stable tie-breaker.
- I built payloads for intermediate leaderboard and final podium.

WHAT I LEARNED:
- Ranking is not just sorting; tied players need consistent rank assignment.
- The client should receive rank, nickname, score, and per-question delta so the

- Final podium can be derived from the full leaderboard instead of maintained as
  separate state.

CODE SNIPPET USED:

                .thenComparing(s -> s.nickname.toLowerCase(Locale.ROOT)))
        .toList();

ISSUES ENCOUNTERED:
- Players with the same score needed deterministic ordering.
- The design document expected a `winners` array and a `sessionSummary` object.
- The app needed to close the session after the final question.

- I added `leaderboardSnapshot()` to generate sorted rank rows.
- I added `finalResultsEvent()` to broadcast winners and session summary.
- I updated `game_sessions.status` to `FINISHED` when the final question closes.
