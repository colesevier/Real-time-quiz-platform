================================================================================

PROMPT 6:
"How do I structure in-memory data to handle concurrent access when dozens 
of players are joining lobbies and submitting answers simultaneously?"

AI RESPONSE SUMMARY:
- Use ConcurrentHashMap for the top-level collection
- Add fine-grained locks (ReentrantLock) for per-lobby/per-game mutations
- Copy-on-read for snapshot consistency (List.copyOf)
- Avoid synchronized blocks; use explicit locks

WHAT I LEARNED:
- ConcurrentHashMap allows concurrent reads; mutations still need locking
- Per-resource locking (per-lobby, per-game) scales better than global lock
- Immutable snapshots prevent iterator exceptions

        lobby.lock.unlock();

ISSUES ENCOUNTERED:
- Without locks, nickname uniqueness check and insert had a race condition
- Large lobby rosters were slow to copy

SESSION 5: Scoring Formula and Leaderboard Calculation
================================================================================

PROMPT 7:
"How do I implement the scoring formula where faster responses earn more 
points, but correctness is mandatory?"

AI RESPONSE SUMMARY:
- Formula: Correctness * 1000 * (1 - (Tsubmit - Tstart) / Total * 0.5)
- Correctness is 1 (right) or 0 (wrong)
- Time penalty maxes out at 50% (even instant submission gets 500 if wrong = 0)
- Server uses server-side clock, not client time

WHAT I LEARNED:
- Time-based scoring incentivizes quick thinking without penalizing too much
- Need to store both answer correctness and submit time for calculation
- Leaderboard must be recalculated when question expires (not per-submission)

CODE SNIPPET USED:
```java
private int pointsFor(CurrentQuestion current, Instant submittedAt) {
    long elapsedMs = Math.max(0, Duration.between(current.startedAt, submittedAt).toMillis());
    long totalMs = Math.max(1, Duration.between(current.startedAt, current.endsAt).toMillis());
    double elapsedRatio = Math.min(1.0, elapsedMs / (double) totalMs);
    return (int) Math.round(1000.0 * (1.0 - elapsedRatio * 0.5));
}
```

ISSUES ENCOUNTERED:
- First implementation accepted client-provided submission time (cheating vector)
- Leaderboard was calculated per-submission (inefficient)

HOW I FIXED IT:
- Changed to use clock.instant() for all timestamps
- Moved score calculation to question expiry (batch processing)
- Test case: verify 5s submit on 20s question yields 875 points
