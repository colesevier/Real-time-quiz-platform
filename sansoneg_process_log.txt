PROMPT 6:
I noticed an issue with authentication. Help with my user authentication. I tried to create an account, the page told me
that I need a longer password. So I tried to create with longer password, and it said the username has already been taken. 
This is the first time I even try to register.
I asked the AI to help me find and fix the bug.

AI RESPONSE SUMMARY:
This is a real bug, and it comes down to two issues working together.
Issue 1: The form doesn't repopulate fields after a validation error.
Look at register.html — none of the inputs have th:value.
So when the "password too short" error fires, the form comes back completely blank. You have to retype everything. 
If you typed the username even slightly differently the second time, you'd hit a different error than you'd expect.
Issue 2: The register method doesn't catch DataIntegrityViolationException.
If two registrations race, or if the email is already taken, jdbc.update() throws an uncaught exception and the user 
gets a 500 instead of a friendly message.
Here are the fixes:
register.html — repopulate fields on error.
AuthController.java — pass fields back to the model, and catch DB duplicates.

WHAT I LEARNED:
Thymeleaf's th:value is required to repopulate inputs after a validation failure —
without it, every error resets the whole form and the user has to retype everything
Always repopulate the model at the top of a POST handler before any early returns,
so the template always has the values it needs regardless of which error fires
DataIntegrityViolationException is Spring's wrapper for SQL constraint violations —
catching it turns a 500 crash into a friendly error message
Validation errors that silently clear user input cause confusion and can make bugs
look like database problems when they aren't
 
TEST CASES COVERED (from Testing Plan):
TC-AUTH-03: Empty fields rejected with message
TC-AUTH-04: Password too short rejected with message
TC-AUTH-06: Username exceeding max length rejected
TC-AUTH-07: Duplicate email handled gracefully (no 500 error)
