PROMPT 10:

AI RESPONSE SUMMARY:
- Inside Docker, localhost refers to the container itself not the host Mac
- Ollama is running on the Mac at port 11434 but Docker can't reach it via localhost

PROMPT 13:
"Error: generation_failed — Unrecognized token 'To': was expecting JSON String..."

AI RESPONSE SUMMARY:
- Model was adding explanatory text AFTER the JSON array ("To summarize...")
- lastIndexOf('}') correctly found the last brace but the text after it
  was included in the string passed to Jackson
- Fix: check if ] appears after the last } — if yes use it, if no close manually

WHAT I LEARNED:
- LLMs frequently add concluding text after structured output even when
  instructed not to — parsing must always strip trailing content
- The three cases to handle: (1) perfect JSON, (2) truncated JSON missing ],
  (3) complete JSON with trailing text after ]

CODE SNIPPET USED:
```java
int lastBrace = responseText.lastIndexOf('}');
int closingBracket = responseText.indexOf(']', lastBrace);
if (closingBracket != -1) {
    // Array closed properly — take up to and including ]
    responseText = responseText.substring(0, closingBracket + 1);
} else {
    // Truncated or trailing text — close manually after last }
    responseText = responseText.substring(0, lastBrace + 1) + "]";
}
```

PROMPT 14:
"It generated 11 questions when I input 10"

AI RESPONSE SUMMARY:
- LLMs don't reliably follow exact count instructions
- Two-part fix: stronger prompt wording AND hard limit with stream().limit(count)
- Prompt: "Generate EXACTLY %d... Not more, not fewer. EXACTLY %d."

WHAT I LEARNED:
- LLM instructions are probabilistic, not deterministic — always enforce
  constraints in code, not just in the prompt
- stream().limit(count) is a simple guaranteed hard cap regardless of model output
- Repeating the constraint multiple times in the prompt ("EXACTLY %d... EXACTLY %d")
  improves but doesn't guarantee compliance

CODE SNIPPET USED:
```java
// Hard cap regardless of model output
return questions.stream().limit(count).collect(Collectors.toList());
```

SESSION 7: Host Flow 404
================================================================================

PROMPT 17:
"404 error when clicking Host button on game cards"

AI RESPONSE SUMMARY:
- Dashboard game cards posted to /host/start/{gameId} as a path variable
- HostController only had POST /host/start with @RequestParam(required=false)
- Path variable (/host/start/42) and request param (/host/start?gameId=42)
  are different routing patterns — Spring couldn't match the route
- Fix: add a new @PostMapping("/start/{gameId}") with @PathVariable

WHAT I LEARNED:
- @PathVariable reads from the URL path itself (/resource/42)
- @RequestParam reads from query string (?id=42) or form body
- They are not interchangeable — Spring routes them differently
- A 404 (not 400 or 500) means no route matched at all

CODE SNIPPET USED:
```java
@PostMapping("/start/{gameId}")
public String startGame(@PathVariable int gameId,
                        @RequestParam(defaultValue = "20") int timePerQuestion,
                        @RequestParam(defaultValue = "50") int maxPlayers,
                        HttpSession session) {
    UUID userId = (UUID) session.getAttribute("userID");
    if (userId == null) return "redirect:/auth/login";
    GameSession s = sessionService.createSession(gameId, userId, timePerQuestion, maxPlayers);
    return "redirect:/host/" + s.getInviteCode();
}
```

PROMPT 19:
"form.append('count', count) was placed before const form = new FormData()"

AI RESPONSE SUMMARY:
- JavaScript throws ReferenceError if you use a variable before declaring it
  with const/let (temporal dead zone)
- form.append() was called before const form = new FormData()

WHAT I LEARNED:
- const and let are not hoisted like var — they exist in a "temporal dead zone"
  from the start of the block until the declaration is reached
- Reading count and validating inputs should happen before creating FormData
- FormData creation and all .append() calls should be grouped together

CODE SNIPPET USED:
```js
// Wrong order — ReferenceError
const count = parseInt(document.getElementById('upload-count').value);
form.append('count', count);  // form doesn't exist yet!
const form = new FormData();

// Correct order
const count = parseInt(document.getElementById('upload-count').value);
if (!file) { ... return; }
if (!title) { ... return; }
const form = new FormData();  // declare first
form.append('file', file);
form.append('title', title);
form.append('count', count);  // then append
```

PROMPT 20:
"What if teammates are not on Mac?"

AI RESPONSE SUMMARY:
- Windows: Docker Desktop supports host.docker.internal natively — no changes
- Linux: host.docker.internal not supported in plain Docker by default
- Fix for Linux: add extra_hosts to docker-compose.yml
- Ollama install command differs by OS

WHAT I LEARNED:
- host.docker.internal is a Docker Desktop feature — not available in
  plain Docker Engine on Linux
- extra_hosts: host-gateway is the Linux equivalent and is harmless on
  Mac/Windows so it's safe to add for everyone
- Always document OS-specific setup steps when teammates use different systems

CODE SNIPPET USED:
```yaml
# docker-compose.yml — works on Mac, Windows, and Linux
services:
  app:
    extra_hosts:
      - "host.docker.internal:host-gateway"
```
